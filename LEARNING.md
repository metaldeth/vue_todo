# Обучение Vue 3: от основ до Todo List

Пошаговые инструкции, чтобы собрать классический Todo List своими руками и параллельно освоить Vue 3.

---

## Часть 0. Подготовка окружения

Если проект ещё не поднят:

1. В корне выполни: `npm install`
2. Запусти dev-сервер: `npm run dev`
3. Открой в браузере адрес из вывода (обычно `http://localhost:5173`)

Должна открыться пустая страница с приложением Vue — значит, окружение готово.

**TypeScript:** в проекте уже подключён TypeScript. В компонентах используется `<script setup lang="ts">`. Типы можно добавлять по мере изучения: например, `ref<number>(0)` или интерфейс для задачи `interface Todo { id: number; text: string; done: boolean }`. Проверка типов: `npm run typecheck`.

## Часть 1. Основы Vue 3

### 1.1 Реактивность и шаблон

**Цель:** понять, как данные из скрипта отображаются в шаблоне.

**Задание:**

1. Открой `src/App.vue`.
2. В `<script setup>` объяви переменную: `const message = 'Привет, Vue!'`.
3. В `<template>` выведи эту переменную: `{{ message }}`.
4. Сохрани и проверь в браузере. Потом замени значение `message` на другое — страница обновится сама (реактивность).

**Что изучить:** [Reactivity Fundamentals](https://vuejs.org/guide/essentials/reactivity-fundamentals.html) — как работают `ref` и реактивные данные.

---

### 1.2 Реактивные данные с `ref`

**Цель:** использовать `ref` для данных, которые будут меняться.

**Задание:**

1. В `App.vue` добавь импорт: `import { ref } from 'vue'`.
2. Создай счётчик: `const count = ref(0)`.
3. В шаблоне выведи `{{ count }}` — так отобразится число. Чтобы вывести значение ref в шаблоне, пиши просто `count` (Vue сам «разворачивает» ref).
4. Добавь кнопку с `@click="count++"`. При клике число должно увеличиваться.

**Что изучить:** в шаблоне ref автоматически разворачивается; в `<script>` для изменения нужно `count.value++`.

---

### 1.3 Список и `v-for`

**Цель:** вывести список элементов циклом.

**Задание:**

1. Создай массив задач (пока захардкоженный):
   ```js
   const todos = ref([
     { id: 1, text: 'Изучить Vue 3', done: false },
     { id: 2, text: 'Сделать Todo List', done: false },
   ])
   ```
2. В шаблоне выведи список через `v-for`:
   ```html
   <ul>
     <li v-for="todo in todos" :key="todo.id">
       {{ todo.text }}
     </li>
   </ul>
   ```
3. Убедись, что у каждого элемента есть уникальный `:key` (здесь `todo.id`).

**Что изучить:** [List Rendering (v-for)](https://vuejs.org/guide/essentials/list.html), зачем нужен `key`.

---

### 1.4 Условный вывод и `v-if` / `v-show`

**Цель:** показывать разный контент в зависимости от условия.

**Задание:**

1. Добавь переменную `const hasTodos = ref(true)`.
2. Над списком выведи сообщение «Список пуст» только когда задач нет. Используй `v-if="todos.length === 0"` (позже, когда добавишь добавление/удаление, список сможет стать пустым).
3. Рядом с текстом каждой задачи выведи «(выполнено)», если `todo.done === true`, через `v-if` или `v-show`. Поэкспериментируй с обоими и почувствуй разницу (v-if убирает из DOM, v-show скрывает через CSS).

---

### 1.5 Обработка событий и методы

**Цель:** вызывать методы по клику и менять состояние.

**Задание:**

1. Объяви метод в `<script setup>`:
   ```js
   function toggleTodo(id) {
     const todo = todos.value.find(t => t.id === id)
     if (todo) todo.done = !todo.done
   }
   ```
2. В списке сделай так, чтобы по клику на задачу вызывался `toggleTodo(todo.id)` (используй `@click`).
3. Добавь визуальное отличие выполненных задач (например, класс или зачёркнутый текст через `:class` или стили по `todo.done`).

**Что изучить:** [Event Handling](https://vuejs.org/guide/essentials/event-handling.html), [Methods](https://vuejs.org/guide/essentials/reactivity-fundamentals.html#declaring-methods).

## Часть 2. Добавление и удаление задач

### 2.1 Поле ввода и новая задача

**Цель:** вводить текст и по кнопке/Enter добавлять задачу в список.

**Задание:**

1. Создай `const newTodoText = ref('')`.
2. В шаблоне добавь `<input v-model="newTodoText" />`. Убедись, что при вводе значение в приложении меняется (двустороннее связывание).
3. Добавь кнопку «Добавить» и метод `addTodo()`:
   - если `newTodoText.value.trim()` пустой — не добавлять;
   - иначе создать объект `{ id: Date.now(), text: newTodoText.value.trim(), done: false }`, добавить его в `todos.value` и очистить `newTodoText`.
4. Сделай так, чтобы по нажатию Enter в поле тоже вызывался `addTodo()` (например, `@keyup.enter="addTodo"`).

**Что изучить:** [v-model](https://vuejs.org/guide/essentials/forms.html), [Form Input Bindings](https://vuejs.org/guide/essentials/forms.html).

---

### 2.2 Удаление задачи

**Цель:** удалять задачу из списка.

**Задание:**

1. Добавь метод `removeTodo(id)`, который удаляет из `todos.value` элемент с данным `id` (например, через `filter` или поиск индекса и `splice`).
2. Рядом с каждой задачей в списке добавь кнопку «Удалить» с `@click="removeTodo(todo.id)"`.
3. При желании добавь подтверждение перед удалением (например, `confirm()`).

## Часть 3. Компоненты (по желанию — следующий уровень)

**Цель:** вынести часть логики и разметки в отдельные компоненты.

**Задание:**

1. Создай папку `src/components/`.
2. Создай компонент `TodoItem.vue`:
   - принимает задачу через `defineProps` (например, `todo`);
   - при клике по задаче вызывает `emit('toggle', todo.id)`;
   - при клике по «Удалить» вызывает `emit('remove', todo.id)`.
3. В `App.vue` используй `<TodoItem>` в `v-for`, передавай `:todo="todo"` и обрабатывай `@toggle="toggleTodo"` и `@remove="removeTodo"`.
4. (Опционально) Вынеси поле ввода и кнопку «Добавить» в компонент `TodoInput.vue` с `emit('add', text)`.

**Что изучить:** [Components Basics](https://vuejs.org/guide/essentials/component-basics.html), [Props](https://vuejs.org/guide/components/props.html), [Emits](https://vuejs.org/guide/components/events.html).

## Часть 4. Улучшения (по желанию)

- Фильтр: «Все / Активные / Выполненные» — один массив `todos`, отображаемый список считай через `computed` в зависимости от выбранного фильтра.
